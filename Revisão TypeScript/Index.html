Revisão TypeScript

Conceitos básicos iniciais

O que é: O TypeScript é um superset para a linguagem JavaScript, ou seja, adiciona funções ao JavaScript, como a declaração de tipos de variável. Ele pode ser adicionado com frameworkd/libs, como Express e React, ou seja, ele também adiciona essa funções a mais no React e no Express.

Por que utilizar: Ele adiciona confiabilidade ao programa, com ele nós temos certeza que uma variável vai passar um certo tipo de dado, ele também provê novas funcionalidades a JS, como Interfaces, ele também deixa o JavaScript mais explícito, diminuindo a quantidade de bugs.

Criando um programa com TS: Devemos criar o arquivo com a extensão ".ts", fazermos as alterações precisamos usar o comando tsc index.ts no terminal, ele ira gerar um arquivo de mesmo nome com a extensão ".js", é esse arquivo que devemos chamar no HTML. Para declarar um tipo da variável podemos adicionar dois pontos na frente da variável e o tipo: function greeting(name: string)

Fundamentos

O que são tipos: A principal função do TypeScript é determinar tipos para os dados, existem vários tipos, como os tipos primitivos, os tipos primitivos (como number, string e boolean).

Type annotation e Type inference: São dois tipos de determinar o tipo, o annotation é quando definimos o tipo manualmente, o inference é quando criamos a variável e ao adicionar o valor o próprio TS define o tipo.

Arquivo de configuração: O TS pode ser configurado de muitas maneiras, mas para isso precisamos do arquivo de configuração, para criar ele utilizamos o tsc --init

Compilar o TS automaticamente: Podemos usar o comando tsc -w para compilar o TS automáticamente (graças a Deus).

Avançando em TS

Arrays: Podemos determinar um tipo de valores de uma lista/vetor: number[]; string[]

Outra sintaxe de arrays: A forma anterior é a mais usada, mas existe outra: const nums: Array<number> = [100, 200, 300]; name: Array<string>

any: O any faz com que uma variável do TS possa ter qualquer tipo de variável (indo contra a idéia inicial de se usar o TS): const arr1: any = [1, "teste", true, [], {nome: "Alexandre"}];

Tipo de parâmetro de funções: Podemos definir o tipo de cada parâmetro de uma função: function soma(a: number, b: number). Podemos definir também tipos para retornos: function greetingName(name: string): string

Funções anônimas: O TS pode ajudar nas funções anônimas, fornecendo dicas de possíveis problemas.

Tipos de objetos: Podemos determinar quais tipos as propriedades de um objeto possuem.

Propriedades opcionais: Nem sempre um objeto possui todas as propriedades que ele poderia possuir, o que pode ser um problema pois o TS vai incomodar até ele ter todas as propriedades, para resolver isso usamos as propriedades opcionais (o primeiro argumento não pode ser opcional): function showNumbers(a: number, b: number, c?:number)

Validação de props opcionais: Quando a propriedade é opcional precisamos criar uma validação, pois o TS não nos ajuda mais.

Union types: É uma alternativa melhor do que o any, com ele podemos determinar mais de um tipo para a variável: function showBalance(balance: string | number)

Condicionais no union type: Quando não sabemos o tipo de dado que vai vir, precisamos utilizar eles de maneira diferente, e para isso usamos uma condicional.

Type alias: É um recurso que permite criar um tipo, e determinar o que ele verifica: type ID = string | number; function showId(id: ID)

Interfaces: É uma outra maneira de nomear o tipo de um objeto, podemos determinar um nome para o tipo, e podemos também escolher quais as propriedades e seus tipos: interface Point{ x: number y: number z: number }

Diferença entre type alias e interfaces: Na maioria das vezes podemos escolher qual das duas usar sem problemas, mas a maior diferença é que a interface pode ser alterada ao longo do código, o alias não.

Literal types: É um recurso que permite colocar valores como tipos, isso restringe o uso não só a tipos, mas também valores, é muito usado junto do union types: function showDirection(direction: "left" | "right" | "center")

Non-null Assertion Operator: As vezes o TS pode acusar um erro baseado em um valor que naquele momento do código ainda não esta disponível, isso acontece muito com DOM, porém se sabemos que o valor será preenchido podemos evitar o erro, para isso usamos o "!": console.log(p!.innerText);

Algum parágrafo!

Bigint: Podemos usar ele para declarar valores muito altos, para usar esse recurso precisamos mudar a configuração do TS para a versão mínima de ES2020: let n: bigint; n = 1000n;

Symbol: Assim como o bigint o symbol precisa de uma versão mais moderna do TS (ES2020), ele cria uma referência única para um valor, ou seja, mesmo que ele possua um mesmo valor de outra variável, teremos valores sendo considerados diferentes: let symbolA:symbol = Symbol("a")
